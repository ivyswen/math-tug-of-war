01 · Executive Summary

执行摘要

Math Tug of War 是一款面向课堂、训练营和现场互动活动的 双人数学竞速游戏。操作者在开局前只需完成 3 步配置，系统就会自动开始倒计时、发题、计分、推动绳索，并在任一队伍达到胜利条件后展示获胜动画与结果面板。

核心 Takeaways

  • 支持 4 类运算 组合出题：加、减、乘、除，可按课堂目标自由启用。
  • 支持 3 档难度：简单为 1-10，中等为 10-50，困难为 30-100。
  • 默认胜利条件为 20 题抢先答对，或绳索位移达到 300 像素 极限。
  • 每次答对推进 50 像素，答错则按反方向回退约 16.7 像素。

02 · Quick Start

快速开始

如果你第一次接触本项目，按本章顺序执行即可在本地浏览器中跑起来并进入可用状态。

适用场景

本项目适合需要 双队对抗、大屏展示 和 快速反复开局 的教学或活动现场。典型使用方式包括小学数学竞赛、训练营暖场、机构公开课互动和展台引流。

启动步骤

  1. 进入项目目录 math-tug-of-war。
  2. 首次运行安装依赖：npm install。
  3. 启动开发服务器：npm run dev -- --host 127.0.0.1 --port 4174。
  4. 浏览器打开 http://127.0.0.1:4174/。
npm install
npm run dev -- --host 127.0.0.1 --port 4174

首次进入你会看到什么

区域 用途 操作者需要关注什么
顶部标题区 显示游戏标题与语言切换 确认当前语言是否正确，必要时切到中文或英文
中央配置弹窗 比赛开始前的三步设置 依次选择运算、难度、队名
左右答题面板 双方题目、输入框和数字键盘 比赛开始后只需在这里输入并提交答案

03 · Gameplay

游戏流程与界面操作

整场比赛从配置到结算是线性的，主持人不需要切页面，也不需要手动发题。

开局流程

步骤 1：选择运算类型

可同时启用一项或多项运算。至少要保留 1 项 运算，系统才允许进入下一步。适合复习单一知识点时只开一种运算，做综合对抗时同时开启多种运算。

步骤 2：选择难度

难度只影响随机数范围，不改变核心玩法。简单适合低年级热身，中等适合常规课堂竞赛，困难更适合训练营或熟练度较高的队伍。

步骤 3：填写队伍名称

左右两队分别对应 蓝方 与 红方。队名会同时显示在答题面板和中央比分栏中，便于大屏观众识别。

开始比赛

点击 START GAME 后，系统会执行一轮 3 秒倒计时，同时尝试播放背景音乐。倒计时结束显示 GO，随后双方答题区解锁并各自生成第一道题目。

答题面板操作

控件 作用 说明
数字按钮 0-9 逐位输入答案 输入框为只读，必须通过屏幕键盘录入
清除按钮 删除当前输入 适合输错后立即重输
提交按钮 校验本次答案 提交期间当前队伍会短暂锁定，避免重复点击
题目显示区 展示当前题干 系统会自动缩放字号，尽量避免长题目溢出

中央竞技区说明

  • 顶部左、右两侧分别显示双方累计答对数量。
  • 中央时钟从 00:00 开始累计，用于活动记录和赛后展示。
  • 绳索向左移动代表蓝方占优，向右移动代表红方占优。
  • 比赛结束后中央视频区域切换为对应胜方动画，覆盖原主场景。
主持建议
正式开始前，建议先做一次空场演练：确认语言、音乐、队名显示、大屏比例、全屏按钮和音量按钮都能正常工作，再让参赛者上场。
04 · Rules

规则、题目与胜负判定

本章覆盖使用者最关心的三件事：题目怎么出、分数怎么算、什么时候算赢。

题目生成规则

难度 随机数范围 适用建议
Easy 1-10 低年级、暖场、首次试玩
Medium 10-50 常规课堂竞赛
Hard 30-100 训练营、进阶挑战

不同运算使用不同出题策略。减法会自动交换两个数字顺序，避免出现负数结果；除法会先生成商，再反推被除数，因此题目默认保证能整除，不会要求玩家输入小数。

计分与绳索位移

动作 分数变化 绳索变化 现场表现
答对 当前队伍 +1 向本队方向移动 50 像素 输入框短暂高亮为正确状态，并立即生成下一题
答错 不加分 向对手方向回退约 16.7 像素 输入框短暂抖动并清空，随后可继续作答

胜利判定

  1. 任一队伍率先答对 20 题，立即获胜。
  2. 如果绳索累计位移达到 300 像素 边界，也立即结束比赛。
  3. 若没有分数先到 20，则以绳索最终方向决定胜方：左侧为蓝方胜，右侧为红方胜。
这套规则兼顾了「持续答对的稳定优势」和「短时间连续抢答带来的视觉冲击」。分数决定明确，位移决定气氛，二者叠加使比赛更适合公开展示。 - 基于 `src/lib/gameEngine.js` 与 `public/game/arqon/script.js` 的默认配置整理
05 · Operations

资源替换、部署与常见问题

如果你不只是在现场使用，而是需要长期维护、定制素材或重新打包，请直接从本章开始操作。

替换视频资源

保持文件名不变，直接替换以下素材即可：

  • public/game/assets/media/main.mp4：主场景循环视频
  • public/game/assets/media/win_blue.mp4：蓝方胜利视频
  • public/game/assets/media/win_red.mp4：红方胜利视频

替换音频资源

默认音频列表定义在 src/config/audioPanelConfig.js。若仅替换文件内容，保持以下路径不变即可：

  • public/game/assets/audio/audio.mp3
  • public/game/assets/audio/audio-1.mp3

构建与发布

npm run build
npm run preview
npm test

构建产物输出到 dist/。如果需要桌面打包，可继续使用 pake:build 或 pake:linux 脚本，将静态页面封装为桌面程序。

常见问题

问题 可能原因 处理建议
点击开始后没有声音 浏览器拦截自动播放 先与页面发生一次交互，或通过右下角静音按钮重新启用音频
题目看起来过长 大数字或乘除组合导致宽度增加 系统已内置字号自适应，仍不合适时建议降低难度或缩短显示区域外部缩放
想改默认语言 页面初始化为英文 检查 src/hooks/useLegacyGameBoot.js 和 public/game/arqon/i18n.js 的默认值
想新增第三首音乐 仅替换文件不够 需要在 src/config/audioPanelConfig.js 中新增一条音频配置
Call to Action
如果这份指南要交给一线教师或活动执行人员使用，建议再补一页「赛前检查单」：网络是否关闭、浏览器是否全屏、音量是否合适、语言是否正确、视频是否加载完毕、备用设备是否到位。
